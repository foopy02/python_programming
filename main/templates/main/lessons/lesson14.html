Ерекшелік дегеніміз не?

Сіз Мәтіндік редактор бағдарламасын әзірледіңіз делік. Бағдарламада пайдаланушы жаңа файл жасай алады, оған деректерді енгізеді, содан кейін файлды жүйеде сақтай алады.


Егер код дұрыс жазылса, қателер болмайды. Бірақ пайдаланушы редакторды ашып, қажетті файлды ашты, оған деректерді жазды, содан кейін файлды компьютерден қолмен жойды, содан кейін файлды сіздің бағдарламаңыз арқылы сақтауға тырысты деп елестетіп көрейік. 


Бұл сценарийде сіз бағдарламаны бұзатын және пайдаланушының бағдарламаңызда жұмыс істеуге деген ықыласын кетіретін қатені аласыз. 


Ерекшелік бағдарламаның өзі жұмыс істеген кезде пайда болатын қате болып табылады. Шартты мәлімдемелер арқылы мұндай қателерді қадағалау әрқашан мүмкін бола бермейді, өйткені бағдарлама қазірдің өзінде жұмыс істеп тұр, біз пайдаланушының әрекетін алдын ала білмейміз, яғни оны жасау кезінде қатені «ұстап алу» керек.


Қателерді ұстау

Жоғарыдағы жағдай үшін қателерді жасалған сәтте анықтау үшін try except құрылымы қажет. 


Бұл құрылымның көмегімен әртүрлі пішімдер мен оқиғалардың қателерін бақылай аласыз. Сіз файлдың қате ашылуын бақылай аласыз, сыныптың қате қосылымын қадағалай аласыз, айнымалының немесе функцияның қате жазылуын қадағалай аласыз және онда не бар, тіпті сандарды нөлге бөлуді бақылай аласыз!


try except construct

іске асырылуы

Қатені бақылауды қосу үшін келесі кодты жазуға болады:

try: # Пробуем что-то сделать
	k = 1 / 0 # Деление на ноль
except ArithmeticError: # Отслеживаем на арифметическую ошибку
	print ("Найдена ошибка") # Мы нашли ошибку
	k = 0 # Наша переменная теперь будет равна 0

print(k) # Просто выводим переменную

Жоғарыдағы код ерекше жағдайларды пайдаланудың өте қарапайым мысалын көрсетеді. Оның қарапайымдылығын елемеңіз, өйткені ең бастысы try except block-ның жалпы құрылысын қарастыру.


Бұл конструкцияны пайдаланған кезде, бағдарлама барысында қандай қате орын алса да, бағдарламаның өзі жұмысын тоқтатпайды және пайдаланушы оны пайдалануда жағымсыз тәжірибе алмайды.


Егер бірнеше қате класын қадағалау қажет болса, блоктардан басқа бірнеше қосуға болады. Олардың барлығы шартты операторлардағы сияқты бірінің астына қосылады.
