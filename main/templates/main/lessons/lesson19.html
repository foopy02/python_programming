Мұрагерлік, инкапсуляция және полиморфизм дегеніміз не, біз қазірдің өзінде талқыладық алдыңғы сабақ, сондықтан біз әзірше оған тоқталмаймыз.


Мұрагерлік

Мұрагерлік - OOP негізгі ұғымдарының бірі. Мұрагерлік арқылы бір жалпы сыныпты (ата-аналық сынып) жасауға және негізгі сыныптан барлық өрістерді, әдістерді және конструкторларды иеленетін көптеген басқа сыныптарды (мұра сыныптарын) жасауға болады.


Мұрагерлік не үшін қажет?

Бізде бір үлкен Көлік класы бар делік. Класс барлық көлік құралдарының негізгі сипаттамаларын сипаттайды:

  • өрістер: жылдамдық, салмақ, қуат қоры және т.б.;
  • әдістер: өрістерден ақпарат алу, жаңа мәндерді орнату;
  • конструктор: бос және барлық өрістерді орнату арқылы.


Сынып негізінде біз жеңіл автокөлік нысанын, жүк көлігі нысанын, ұшақ нысанын және т.б. жасай аламыз. Барлық нысандардың сипаттамалары мен әдістері бірдей болады. 


Біз көлік пен ұшақ нысанының өрістері мен сипаттамалары әртүрлі болатынын анық түсінеміз. Мұны қалай істеу керек:

  1. Сіз екі бөлек сыныпты жасай аласыз: "Автомобиль" және "Ұшақ". Әрбір сыныпта Transport сыныбынан қайта жазылған барлық әдістер, өрістер және конструкторлар, сонымен қатар белгілі бір сынып үшін ғана маңызды жаңа әдістер болады;
  2. Сіз екі бала сыныбын жасай аласыз: "Автомобиль" және "Ұшақ". Екі класс «Транспорт» сыныбынан барлығын мұраға алады және сонымен бірге өздерінің қосымша функцияларын қамтиды. Осылайша, кодтың қайталануы болмайды және код кішірейіп, таза болады.


Ұрпақ кластарын жасау

Мұрагер класс жасау үшін класс жасап, негізгі сыныптан мұраны көрсету керек.


Ұрпақ класының мысалы:

class Cars:
	wheels = 4 # Общее значение для всех объектов,
	# так как все машины имеют колеса
		
class BMW (Cars):
	is_m_serias = true # Является ли модель "М" серии?
	# Переменная нужна только в классе BMW


Инкапсуляция

Инкапсуляция сыныптағы функцияға қол жеткізуді шектеуге мүмкіндік береді. Осы тәсілдің арқасында шабуылдаушылар немесе өзіміз байқаусызда немесе әдейі қоңырау шала алмаймыз немесе әдісті өзгерте алмаймыз.


Мысалы:

class Some:
	def _printWords(self):
		print ("Попробуй меня вызвать")

obj = Some()
obj._printWords() # Вызов функции ничего не даст

Астын екі рет сызу арқылы қорғаныс әсері күшейтіледі, сондықтан функцияны шақыру мүмкін болмайды.


Полиморфизм

Полиморфизм ұрпақ кластарындағы функцияларды өзгертуге мүмкіндік береді. Мысалы:
class Cars:
	def nothing(self): # Пустая функция
		pass
		
class BMW (Cars):
	def nothing(self, word):
		print (word + "!") # Функция теперь будет работать по новому
 
a = BMW()
a.nothing("Some")
